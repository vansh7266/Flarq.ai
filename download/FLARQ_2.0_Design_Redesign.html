Section 01
Design Philosophy & Color System
FLARQ is not a SaaS dashboard. It is an AI operating system. The design must feel like a command center, not a spreadsheet. Dark-first, ambient, and alive.
Three Design Principles
1
Dark Command Center
Dark mode is the default. Deep space blacks with luminous accents create focus and reduce eye strain during long job search sessions. Light mode exists but dark is identity.
2
Ambient Intelligence
The UI breathes. The Flarq Orb pulses gently, backgrounds shift with context, and micro-animations signal that the AI is always thinking. The product never feels static.
3
Neural Aesthetics
Gradients flow like synapses. Tool executions appear as glowing nodes on a graph. The design language borrows from neural network visualizations, not corporate dashboards.

Why This Palette Wins
Indigo → Sky gradient is the primary identity. It evokes neural networks, deep space, and AI-native products. Unlike the old teal (#0d9488) which reads as "healthcare" or "finance," indigo reads as "intelligence."

Amber is the CTA color. It creates warmth and urgency against the cool dark background. "Start for free" and "Analyze" buttons in amber pop like a beacon.

Emerald signals success (match scores, offers, completed). Rose signals gaps, alerts, and missing skills. Together they create a semantic color language that is instantly scannable.

Section 03
Animation System
Animations in FLARQ 2.0 serve a purpose: they communicate state, guide attention, and create the feeling that the AI is alive. Nothing moves without reason.
1. The Flarq Orb Hero Element
The Orb is the visual embodiment of the AI. It appears on the Agent page, landing page, and as a micro-icon in the navbar. It has three states:
Idle State
Gentle breathing pulse (scale 1.0 → 1.03, 4s ease-in-out). Three orbiting dots at different speeds (6s, 10s, 14s). Box-shadow pulses from 0.3 → 0.5 opacity.
Thinking State
Orbiting dots accelerate (2s, 3.5s, 5s). Orb glows brighter (shadow 0.5 → 0.8). Ring around orb rotates with a conic-gradient shimmer. Dots leave trailing afterimages.
Responding State
Orb returns to idle but a subtle ripple emanates outward (expanding ring, 2s, fade out). This signals that a response has been delivered.

Core Animations
fadeUp
Elements enter from below with opacity 0→1 and y: 20→0. Duration 0.5s, ease-out. Used for page sections, cards, and list items. Stagger: 0.08s between siblings.
scaleIn
Match score ring, modals, and toast notifications scale from 0.9→1 with opacity. Spring physics: stiffness 260, damping 20. Creates a satisfying "pop" on arrival.
shimmer
Skeleton loaders use a horizontal gradient sweep (transparent → rgba(255,255,255,0.05) → transparent) that moves left to right over 2s. Replaces the old "pulse" skeleton.
drawIn
SVG stroke-dashoffset animation for the landing page NodeGraph and decorative icons. Lines draw from 0 to full length over 1.5s with ease-out. Creates a "blueprint being sketched" effect.
countUp
Dashboard stats animate from 0 to their value over 1.2s with ease-out. Uses useCountUp hook with requestAnimationFrame. Numbers tick up in a satisfying, precise way.
ripple
When the agent responds, a ring expands outward from the Orb (scale 1→2.5, opacity 0.6→0, 2s). Creates a "sonar ping" effect that signals the AI has finished thinking.
// framer-motion variants
const fadeUp = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0,
    transition: { duration: 0.5, ease: "easeOut" } },
}
const stagger = {
  animate: { transition: { staggerChildren: 0.08 } },
}
const scaleIn = {
  initial: { scale: 0.9, opacity: 0 },
  animate: { scale: 1, opacity: 1,
    transition: { type: "spring", stiffness: 260, damping: 20 } },
}

Auth Page
Visual: Split layout. Left half: dark surface with the Flarq Orb breathing in the center, tagline "Your AI career strategist" in gradient text. Right half: auth form on a slightly lighter card surface.

Dashboard Page
Visual: Dark command center feel. Time-based greeting with user's first name in gradient text ("Good morning, Sarah").

Key changes:
- Stat cards use GlowCard with gradient borders (Neural, Sunset, Horizon, Aurora — one per card)
- Each stat has a trend indicator (arrow + percentage change)
- Add a sparkline chart in each stat card (tiny AreaChart, 7 data points)
- Quick action cards have unique subtitles and matching gradient icons
- Recent applications section with hover-expand rows
- Stale applications banner uses rose gradient with pulsing dot
Profile Page
Visual: Full-width dark page with a profile header card at the top.

Key changes:
- Profile avatar: Circle with user's initials on a Neural gradient background (replaces plain letter in navbar)
- Profile completeness bar: Animated progress with Horizon gradient fill, percentage label
- Skills section: Inline tag input with autocomplete. Type to add, click X to remove (the remove button now works!). Tags use the 5-color system (purple=technical, sky=framework, emerald=matching, rose=gap, amber=tool)
- Experience timeline: Each entry is a GlowCard with a left border accent (3px gradient line)
- Resume upload: Drop zone with dashed border that transitions to a solid gradient border on drag-over. Animated upload progress with a shimmer effect
- Manual edit mode: Slide-in panel from the right instead of an inline card
Sidebar Navigation New
Replace the top Navbar (authenticated pages only) with a dark sidebar on the left. The sidebar contains: Flarq Orb logo at top, icon + label nav items, user avatar at bottom.

Why: Dashboard-heavy apps need persistent navigation. A sidebar is more scannable than a top bar, provides more vertical space for content, and creates a professional "app" feel. The sidebar is 64px wide (icons only) by default, expanding to 240px on hover or on wider screens.

Agent Page — Complete Redesign Blueprint
1
Layout: Full-Height Dark Canvas
No PageWrapper, no Navbar. The entire viewport is a dark canvas (bg-void #07080a). A floating sidebar on the left (240px) shows conversation history. The Flarq Orb sits at the top of the sidebar, always visible, always breathing. The sidebar background is slightly lighter (bg-surface) with a gradient right border.
2
The Orb: Always Present, Always Alive
The Flarq Orb (64px) sits in the top-left of the sidebar. In idle state, it breathes with a gentle pulse. When the user sends a message, it transitions to "thinking" state — orbiting dots accelerate, glow intensifies, a conic-gradient ring spins around it. When the response arrives, a ripple emanates outward. The Orb NEVER disappears. It is the visual anchor of the entire page.
3
User Messages: Gradient Bubbles
User messages use a Neural gradient background (indigo→sky, darker variant) with white text. They appear on the right side with rounded corners (bottom-right: 4px). This creates a strong visual distinction from the old flat teal bubbles.
4
Agent Messages: Markdown + Ambient Glow
Agent messages are rendered with react-markdown (supports bold, lists, code blocks, headers). Each message has a subtle ambient glow behind it (absolute inset -8px, bg-primary/5, blur-xl). The Flarq Orb avatar (24px mini version) appears to the left of each message. Messages enter with fadeUp animation, staggered 0.03s per message.
5
Tool Execution: Live Node Graph
When the agent calls a tool, a chip appears inline with a glowing icon and label ("Searched 12 applications", "Ran analytics pipeline"). On hover, the chip expands to show a mini card with the tool result preview. The chips use semantic colors: sky for search, amber for analytics, emerald for profile, rose for alerts. This is the "neural aesthetics" principle in action.
6
Input Bar: Floating with Glow
The input bar is fixed at the bottom, centered (max-width 720px). It has a gradient border (Neural) that glows on focus. The send button is a circle with the Neural gradient that glows on hover (box-shadow expands from 0 to 20px). Suggestion chips appear above the input as small, clickable pills with ghost styling. The typing indicator is replaced by the Orb's "thinking" state animation.

Section 10
Light Mode & Implementation Priority
Light Mode (Secondary)
Dark mode is the identity, but light mode is required for accessibility. The light palette inverts the surfaces while keeping the accent colors identical:

Void → White (#ffffff) | Surface → #f8fafc | Card → #ffffff | Border → #e2e8f0 | Text → #0f172a | Muted → #64748b

The Orb, gradients, and accent colors remain the same. GlowCard borders use the same gradients but at 30% opacity (reduced from 40% in dark mode). The Agent page keeps its dark theme even in light mode — it is a "dark room" within the app.
